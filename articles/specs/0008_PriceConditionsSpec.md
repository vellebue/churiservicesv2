# Price Conditions

This spec is used to describe price conditions that are used to determine the final price of an article item depending on the
context of a certain order where this price should be issued.

## Entities

The set of entities to manage price conditions are described below:

- PriceCondition represented in database with table PRICE_CONDITIONS

  - conditionId: Long primary key generated by sequence SEQ_PRICE_CONDITIONS.

  - conditionType: 4 chars, required, references a condition type as described below by PriceConditionType entity.

  - scope: Char, required, describe the scope for this discount condition:
    - 'L' : Line condition, it is applied for an order item that references this price condition.
    - 'H' : Heading condition, it is applied for the whole order.
    - 'D' : Equipment allowance deduction, to be ignored on sale process, retained for accountancy process.

  - evaluation: Char, required, describe the strategy to eval this price condition:
    - 'D': Disjoint strategy. The condition is considered if it is present on, at least, one coditions source.
    - 'J': Joint strategy. The condition is considered if it is present on all coditions sources.


- PriceConditionType: represented in database by C_PRICE_CONDITION_TYPES table.

  - conditionType: 4 chars, primary key, is the mnemonic for a certain price condition type.

  - decription: 256 chars, required. It describes the type of condition.

  - key: 100 chars, required, it is the I18N key to describe this condition type on several languages for messages property files.

  The set of constant values for condition types are as below:

| conditionType | description          | key                               |
|---------------|----------------------|-----------------------------------|
| FARE          | Base Article Price   | priceCondition.type.fare          |
| DISC          | Discount             | priceCondition.type.discount      |
| TAXE          | Tax Condition        | priceCondition.type.tax           |
| PROV          | Accounting Provision | priceCondition.type.prov          |


- PriceConditionValue: represented in database by PRICE_CONDITION_VALUES

  - valueId: Long primary key generated by sequence SEQ_PRICE_CONDITION_VALUES.

  - conditionId: Long, required, references the PriceCondition this PriceConditionValue belongs to.

  - type: Char, required, it represents the type of value for this value condition.
    
    - 'A': Amount: this condition value represents an absolute value applied once per article reference unit.

    - 'P': Percentage: this condition value represents a percentage value to apply from a reference previous amount.

  - value: Decimal(12,6), required, represents the reference quantity for this value condition. Use BigDecimal type to handle in Kotlin. It could be positive or negative.

  - beginDate: Date, required, it represents the begining date of validity for this condition value.

  - endDate: Date, optional, it represents the ending date of validity for this condition value.

  For a certain set of price conditions values related to a single PriceCondition, the overlapping of beginDate-endDate intervals is not permitted.
